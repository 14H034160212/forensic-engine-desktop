Forensic Engine — how it works

A blind, fully-local document reasoning engine. Nothing leaves the machine.

Step 0 — reader inference + author intent

What a reasonable reader takes away, and what the author appears to want them to believe — before any decomposition.

Steps 1–2 — reasoning atom register

The document is decomposed into its smallest discrete reasoning commitments — explicit and implied claims, assumptions, presuppositions, implications, evidence burdens — each traced to source with a stable ID.

Step 3 — bond mapping

How atoms relate: dependencies, support, and, critically, contradictions — pairs that cannot both hold. This surfaces structure a single pass never sees.

Deterministic arithmetic

Unit-economics figures are pulled from the document and the damaging ratios computed in code — model-independent, so even a small local model recovers them.

Blind by design: it excavates what is inside a document; external fact-checking is a separate layer. Decision-support, not financial advice.
